# HotAlert 开发里程碑

> **原则**：每个里程碑结束后，产出都是一个**有效的产出物**（可编译、可运行、可验证）。

---

## 里程碑 1：项目基础架构

**目标**：搭建项目骨架，建立开发基础

- [x] 创建 WPF 项目结构（.NET 6+）
- [x] 建立目录结构（Views、ViewModels、Services、Models、Resources）
- [x] 实现 MVVM 基础框架
- [x] 实现配置模型 `AppConfig.cs`
- [x] 实现配置服务 `ConfigService.cs`（JSON 读写到 `%AppData%`）

**验收标准**：`dotnet build` 编译通过，无错误无警告

---

## 里程碑 2：核心资源监控

**目标**：实现 CPU/内存使用率的实时监控

- [ ] 实现 `ResourceMonitor.cs` 服务
- [ ] 使用 PerformanceCounter 获取 CPU 使用率
- [ ] 获取内存使用率
- [ ] 3 秒采样间隔的定时器
- [ ] 资源变化事件通知机制

**验收标准**：运行程序后，控制台/调试窗口每 3 秒输出当前 CPU 和内存使用率

---

## 里程碑 3：边框警告显示

**目标**：实现核心的边框警告功能

- [ ] 创建 `BorderOverlay.xaml` 透明覆盖窗口
- [ ] 实现四边边框渐变效果
- [ ] 实现动态宽度计算
- [ ] 实现呼吸灯动画效果
- [ ] 实现 `AlertService.cs` 警告管理
- [ ] 多显示器支持（所有屏幕同时显示）
- [ ] DPI 感知和缩放
- [ ] 鼠标悬浮显示数值提示

**验收标准**：当 CPU 或内存超过阈值时，屏幕四周显示带呼吸动画的渐变边框，鼠标悬浮显示数值

---

## 开发顺序

```
M1 → M2 → M3 → M4 → M5 → M6
```

- **M1-M3** 完成后可看到核心功能演示
- **M4** 完成后达到 MVP 可用状态
- **M5-M6** 完善用户体验和发布准备
